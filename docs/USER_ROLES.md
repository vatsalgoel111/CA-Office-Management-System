# User Roles

## Administrator

Usually the Chartered Accountant or trusted office administrator.

Responsibilities:

- Manage clients
- Manage staff
- Allocate work
- Review work progress
- Track billing and collections
- Review reports
- Manage settings
- Monitor audit logs
- Maintain backups

## Staff

Office users responsible for assigned work.

Responsibilities:

- View assigned work
- Update task status
- Add remarks
- Mark work progress accurately

## Additional Supported Roles

The architecture should support these roles even if Version 1.0 exposes only Administrator and Staff in the UI:

- Manager
- Accountant
- Read Only

For Version 1.0, Manager, Accountant, and Read Only may be seeded internally but hidden from normal setup until needed.

Client portal access is a future feature and should not be mixed with staff login design yet.
