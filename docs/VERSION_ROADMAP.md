# Version Roadmap

This roadmap will be maintained throughout the project.

## Version 1.0 (Current)

Goal: production-ready single-office Windows desktop application using SQLite.

Planned features:

- Login
- Role-based access control foundation
- Dashboard
- Client management
- Work management
- Staff management
- Billing and collection tracking
- Reports
- WhatsApp notifications
- SQLite database
- Backup
- Audit log
- Settings
- Dark mode
- Windows executable packaging

## Version 1.1

Goal: improve office operations after the first production release.

Candidate features:

- Email notifications
- Attendance
- Leave management
- Better reporting
- More granular permissions
- Report templates
- Improved reminder rules

## Future Versions

Goal: evolve into a larger office platform when there is proven demand.

Candidate features:

- PostgreSQL/MySQL
- Local server API
- Mobile app
- Cloud sync
- AI assistant
- Client portal
- Document management
- Multi-branch office support

## Versioning Principles

- Version 1.0 must stay focused and deployable.
- Future features must not make the Version 1.0 desktop app unnecessarily complex.
- Architecture should support future growth without over-engineering the first release.
- Major database or deployment changes require architecture review before implementation.
